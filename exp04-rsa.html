Experiment 04 — Asymmetric Cryptography
RSA Algorithm
A public-key cryptosystem where encryption and decryption use different keys. Security is based on the practical difficulty of factoring the product of two large prime numbers.
AsymmetricPublic-KeyNumber TheoryModular Exponentiation
Aim
To implement RSA key generation, message encryption, and decryption; and to understand how public-key cryptography enables secure communication without shared secrets.
Key Generation Steps
  1. Choose two distinct primes p and q. Compute n = p × q
  2. Compute Euler's totient: φ(n) = (p−1)(q−1)
  3. Choose public exponent e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1
  4. Compute private key d such that e × d ≡ 1 (mod φ(n))
  5. Public key: (e, n) — Private key: (d, n)
Formulas
Encryption
C = M^e mod n
Decryption
M = C^d mod n

def g(a, b):
    while b: a, b = b, a % b
    return a

def inv(e, p):
    for d in range(1, p):
        if e * d % p == 1: return d

def keys():
    p = int(input("p: "))
    q = int(input("q: "))
    n = p * q; phi = (p - 1) * (q - 1)
    e = 2
    while g(e, phi) != 1: e += 1
    return e, inv(e, phi), n

e, d, n = keys()
print("Public:", (e, n))
print("Private:", (d, n))

t = input("Text: ")
c = [pow(ord(ch), e, n) for ch in t]
print("Encrypted:", c)
print("Decrypted:", ''.join(chr(pow(x, d, n)) for x in c))

Conclusion
RSA enables secure key exchange and digital signatures without a pre-shared secret. Its security relies on the integer factorization problem — easy to multiply two primes, computationally infeasible to factor the product for large primes. In practice, RSA uses key sizes of 2048–4096 bits. Quantum computers (Shor's algorithm) could theoretically break RSA, driving interest in post-quantum cryptography.
