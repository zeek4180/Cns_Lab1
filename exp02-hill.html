Hill Cipher
A polygraphic substitution cipher based on linear algebra. Encrypts multiple letters simultaneously using matrix multiplication modulo 26, making it resistant to simple frequency analysis.
SymmetricPolygraphicMatrix MathLinear Algebra
Aim
To implement the Hill Cipher using matrix multiplication and modular arithmetic, and to understand the role of the key matrix inverse in decryption.
Theory
Encryption
C = K · P (mod 26)
Decryption
P = K⁻¹ · C (mod 26)
Key Requirement
Key matrix must be invertible mod 26. det(K) must be coprime to 26.
Block Size
Determined by matrix dimension n. n=2 processes 2 letters at a time; padding with 'X'.
Algorithm
  1. Convert plaintext to uppercase, remove spaces, pad with 'X' to multiple of n
  2. Convert each character to its numeric value (A=0, B=1, ..., Z=25)
  3. Group the numbers into column vectors of length n (block size)
  4. Multiply each block vector by the key matrix K modulo 26
  5. Convert resulting numbers back to characters to obtain ciphertext
  6. For decryption, compute modular inverse of K and repeat multiplication
Source Code
Python 
import numpy as np

def inv(a, m):
    for i in range(1, m):
        if (a * i) % m == 1: return i

def mat_inv(k, m):
    d = int(round(np.linalg.det(k))) % m
    di = inv(d, m)
    if di is None: return None
    adj = np.array([[k[1][1], -k[0][1]], [-k[1][0], k[0][0]]])
    return (di * adj) % m

def t2n(t, n):
    t = t.upper().replace(" ", "")
    while len(t) % n: t += 'X'
    return [ord(c) - 65 for c in t]

def hill(t, k, dec=0, m=26):
    n = len(k)
    if dec: k = mat_inv(k, m)
    if k is None: return "Invalid key"
    nums = t2n(t, n); res = []
    for i in range(0, len(nums), n):
        b = np.array(nums[i:i+n]).reshape(n, 1)
        res += list((k @ b % m).flatten())
    return ''.join(chr(int(i) + 65) for i in res)

k = np.array([[5, 8], [17, 3]])
t = input("Text: ")
e = hill(t, k)
print("Encoded:", e)
print("Decoded:", hill(e, k, 1))

Conclusion
The Hill Cipher provides stronger security than Caesar/Vigenère by encrypting multiple characters simultaneously. Its security depends entirely on the key matrix; an invertible matrix mod 26 is mandatory. It is vulnerable to known-plaintext attacks since the cipher is linear — if n pairs (plaintext block, ciphertext block) are known, the key matrix can be recovered.
