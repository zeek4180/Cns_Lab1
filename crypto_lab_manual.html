Experiment 01 — Classical Cipher
Caesar Cipher
A substitution cipher in which each letter of the plaintext is replaced by a letter a fixed number of positions down the alphabet. Named after Julius Caesar, who used it in his private correspondence.
Symmetric Substitution Monoalphabetic O(n) Time
Aim
To implement and demonstrate the Caesar Cipher encryption and decryption technique, and to understand the concept of substitution ciphers and their vulnerabilities to frequency analysis.
Objectives
  • Understand the concept of shift-based substitution in classical cryptography
  • Implement both encryption and decryption using modular arithmetic
  • Analyze the cipher's vulnerability to brute-force attacks (only 25 possible keys)
  • Observe how case preservation is handled in implementation
Theory
Encryption Formula
E(x) = (x + n) mod 26
Decryption Formula
D(x) = (x − n) mod 26
Key Space
26 possible shifts (0–25). Shift 0 = identity. Trivially broken by brute force.
Historical Use
Julius Caesar used shift 3. ROT13 (shift 13) is a modern symmetric variant.
Algorithm
  1. Accept plaintext input and integer shift value s from the user
  2. Reduce s modulo 26 to normalize (e.g., shift 27 → shift 1)
  3. For each character: if uppercase, apply (ord(c) − 65 + s) % 26 + 65
  4. If lowercase, apply (ord(c) − 97 + s) % 26 + 97; non-alpha chars pass unchanged
  5. Concatenate transformed characters to produce ciphertext
  6. To decrypt, apply the same function with shift −s (or equivalently 26 − s)
Source Code
Python 
def c(t, s):
    return ''.join(
        chr((ord(x) - 65 + s) % 26 + 65) if x.isupper() else
        chr((ord(x) - 97 + s) % 26 + 97) if x.islower() else x
        for x in t
    )

t = input("Text: ")
s = int(input("Shift: ")) % 26
e = c(t, s)

print("Encoded:", e)
print("Decoded:", c(e, -s))

Conclusion
The Caesar Cipher demonstrates the fundamental concept of substitution encryption. While historically significant, it is trivially broken by brute-force (only 25 non-trivial keys) or frequency analysis (letter frequency in ciphertext matches natural language). It serves as the basis for understanding more complex polyalphabetic and transposition ciphers.
Experiment 02 — Classical Cipher
Hill Cipher
A polygraphic substitution cipher based on linear algebra. Encrypts multiple letters simultaneously using matrix multiplication modulo 26, making it resistant to simple frequency analysis.
SymmetricPolygraphicMatrix MathLinear Algebra
Aim
To implement the Hill Cipher using matrix multiplication and modular arithmetic, and to understand the role of the key matrix inverse in decryption.
Theory
Encryption
C = K · P (mod 26)
Decryption
P = K⁻¹ · C (mod 26)
Key Requirement
Key matrix must be invertible mod 26. det(K) must be coprime to 26.
Block Size
Determined by matrix dimension n. n=2 processes 2 letters at a time; padding with 'X'.
Algorithm
  1. Convert plaintext to uppercase, remove spaces, pad with 'X' to multiple of n
  2. Convert each character to its numeric value (A=0, B=1, ..., Z=25)
  3. Group the numbers into column vectors of length n (block size)
  4. Multiply each block vector by the key matrix K modulo 26
  5. Convert resulting numbers back to characters to obtain ciphertext
  6. For decryption, compute modular inverse of K and repeat multiplication
Source Code
Python 
import numpy as np

def inv(a, m):
    for i in range(1, m):
        if (a * i) % m == 1: return i

def mat_inv(k, m):
    d = int(round(np.linalg.det(k))) % m
    di = inv(d, m)
    if di is None: return None
    adj = np.array([[k[1][1], -k[0][1]], [-k[1][0], k[0][0]]])
    return (di * adj) % m

def t2n(t, n):
    t = t.upper().replace(" ", "")
    while len(t) % n: t += 'X'
    return [ord(c) - 65 for c in t]

def hill(t, k, dec=0, m=26):
    n = len(k)
    if dec: k = mat_inv(k, m)
    if k is None: return "Invalid key"
    nums = t2n(t, n); res = []
    for i in range(0, len(nums), n):
        b = np.array(nums[i:i+n]).reshape(n, 1)
        res += list((k @ b % m).flatten())
    return ''.join(chr(int(i) + 65) for i in res)

k = np.array([[5, 8], [17, 3]])
t = input("Text: ")
e = hill(t, k)
print("Encoded:", e)
print("Decoded:", hill(e, k, 1))

Conclusion
The Hill Cipher provides stronger security than Caesar/Vigenère by encrypting multiple characters simultaneously. Its security depends entirely on the key matrix; an invertible matrix mod 26 is mandatory. It is vulnerable to known-plaintext attacks since the cipher is linear — if n pairs (plaintext block, ciphertext block) are known, the key matrix can be recovered.
Experiment 03 — Classical Cipher
Playfair Cipher
A digraph substitution cipher that encrypts pairs of letters using a 5×5 key square. It was the first practical digraph substitution cipher, used by the British military in both World Wars.
SymmetricDigraph5×5 Key SquareI/J Combined
Aim
To implement the Playfair Cipher and understand digraph substitution, key square construction, and the three rules governing encryption of letter pairs.
Theory — 3 Encryption Rules
  1. Same Row: Each letter replaced by letter immediately to its right (wrapping)
  2. Same Column: Each letter replaced by letter immediately below (wrapping)
  3. Rectangle: Each letter replaced by letter on same row but in the column of the other
Source Code
Python 
def sq(k):
    k = "".join(c for c in k.upper() if c.isalpha()).replace("J", "I")
    s = []
    for c in k + "ABCDEFGHIKLMNOPQRSTUVWXYZ":
        if c not in s: s.append(c)
    return [s[i:i+5] for i in range(0, 25, 5)]

def play(t, k, d=0):
    s = sq(k)
    p = prep(t) if not d else [t[i:i+2] for i in range(0, len(t), 2)]
    r = ""
    for a, b in p:
        ra, ca = pos(s, a); rb, cb = pos(s, b)
        if ra == rb:
            r += s[ra][(ca + (-1 if d else 1)) % 5] + s[rb][(cb + (-1 if d else 1)) % 5]
        elif ca == cb:
            r += s[(ra + (-1 if d else 1)) % 5][ca] + s[(rb + (-1 if d else 1)) % 5][cb]
        else:
            r += s[ra][cb] + s[rb][ca]
    return r

k = "PLAYFAIR EXAMPLE"
t = input("Text: ")
e = play(t, k)
print("Encoded:", e)
print("Decoded:", play(e, k, 1))

Conclusion
Playfair is significantly stronger than monoalphabetic ciphers as it encrypts digraphs (letter pairs), making simple frequency analysis ineffective. There are 676 possible digraphs vs 26 letters. However, the 25-letter key square is reconstructible from known plaintext, and modern computers can break it through hill-climbing optimization.
Experiment 04 — Asymmetric Cryptography
RSA Algorithm
A public-key cryptosystem where encryption and decryption use different keys. Security is based on the practical difficulty of factoring the product of two large prime numbers.
AsymmetricPublic-KeyNumber TheoryModular Exponentiation
Aim
To implement RSA key generation, message encryption, and decryption; and to understand how public-key cryptography enables secure communication without shared secrets.
Key Generation Steps
  1. Choose two distinct primes p and q. Compute n = p × q
  2. Compute Euler's totient: φ(n) = (p−1)(q−1)
  3. Choose public exponent e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1
  4. Compute private key d such that e × d ≡ 1 (mod φ(n))
  5. Public key: (e, n) — Private key: (d, n)
Formulas
Encryption
C = M^e mod n
Decryption
M = C^d mod n
Source Code
Python 
def g(a, b):
    while b: a, b = b, a % b
    return a

def inv(e, p):
    for d in range(1, p):
        if e * d % p == 1: return d

def keys():
    p = int(input("p: "))
    q = int(input("q: "))
    n = p * q; phi = (p - 1) * (q - 1)
    e = 2
    while g(e, phi) != 1: e += 1
    return e, inv(e, phi), n

e, d, n = keys()
print("Public:", (e, n))
print("Private:", (d, n))

t = input("Text: ")
c = [pow(ord(ch), e, n) for ch in t]
print("Encrypted:", c)
print("Decrypted:", ''.join(chr(pow(x, d, n)) for x in c))

Conclusion
RSA enables secure key exchange and digital signatures without a pre-shared secret. Its security relies on the integer factorization problem — easy to multiply two primes, computationally infeasible to factor the product for large primes. In practice, RSA uses key sizes of 2048–4096 bits. Quantum computers (Shor's algorithm) could theoretically break RSA, driving interest in post-quantum cryptography.
Experiment 05 — Cryptographic Hash
SHA-1 Hash Function
SHA-1 (Secure Hash Algorithm 1) is a cryptographic hash function that produces a 160-bit (40 hex character) digest. Designed by the NSA and published by NIST in 1995.
One-Way160-bit OutputMerkle–DamgårdDeprecated
Aim
To implement SHA-1 from scratch and understand message padding, compression rounds, and the properties of cryptographic hash functions including avalanche effect and collision resistance.
Algorithm Overview
  1. Pad message: append 0x80, zero-pad until length ≡ 448 (mod 512), append 64-bit message length
  2. Initialize five 32-bit hash values: H0–H4 with fixed constants
  3. Process each 512-bit block: expand 16 words to 80 using XOR + left-rotate-1
  4. Run 80 rounds using three functions (Ch, Parity, Maj) with four round constants
  5. Update H0–H4 by adding the round outputs, producing the final 160-bit digest
Source Code
Python — SHA-1 Implementation 
def r(n, b): return ((n << b) | (n >> (32 - b))) & 0xffffffff

def sha1(m):
    m = bytearray(m, 'utf-8'); l = len(m) * 8
    m += b'\x80'
    while (len(m) * 8) % 512 != 448: m += b'\x00'
    m += l.to_bytes(8, 'big')

    h0,h1,h2,h3,h4 = 0x67452301,0xEFCDAB89,0x98BADCFE,0x10325476,0xC3D2E1F0

    for i in range(0, len(m), 64):
        w = [int.from_bytes(m[i+j:i+j+4], 'big') for j in range(0, 64, 4)]
        for j in range(16, 80): w.append(r(w[j-3]^w[j-8]^w[j-14]^w[j-16], 1))
        a,b,c,d,e = h0,h1,h2,h3,h4

        for j in range(80):
            if j<20:   f=(b&c)|(~b&d); k=0x5A827999
            elif j<40: f=b^c^d;       k=0x6ED9EBA1
            elif j<60: f=(b&c)|(b&d)|(c&d); k=0x8F1BBCDC
            else:       f=b^c^d;       k=0xCA62C1D6
            a,b,c,d,e = (r(a,5)+f+e+k+w[j])&0xffffffff,a,r(b,30),c,d

        h0=(h0+a)&0xffffffff; h1=(h1+b)&0xffffffff
        h2=(h2+c)&0xffffffff; h3=(h3+d)&0xffffffff
        h4=(h4+e)&0xffffffff

    return '%08x%08x%08x%08x%08x' % (h0, h1, h2, h3, h4)

t = input("Text: ")
print("SHA1:", sha1(t))

Conclusion
SHA-1 demonstrates core hash function properties: determinism, avalanche effect (single bit change alters ~50% of output bits), and pre-image resistance. However, SHA-1 is now cryptographically broken — Google demonstrated a practical collision attack (SHAttered, 2017). It has been deprecated for security use; SHA-256 or SHA-3 are recommended for new systems.
Experiment 06 — Cryptographic Hash
MD5 Hash Function
MD5 (Message Digest Algorithm 5) produces a 128-bit hash. Designed by Ronald Rivest in 1991. Widely used for checksums and file integrity verification despite known cryptographic weaknesses.
One-Way128-bit OutputLittle-EndianBroken
Aim
To implement MD5 and compare its structure with SHA-1, understanding the use of sine-derived constants, four auxiliary functions, and the little-endian byte order.
MD5 vs SHA-1 Comparison
Output Size
MD5: 128 bits (32 hex)  |  SHA-1: 160 bits (40 hex)
Rounds
MD5: 64 rounds (4×16)  |  SHA-1: 80 rounds (4×20)
Byte Order
MD5: Little-Endian  |  SHA-1: Big-Endian
Constants
MD5: |sin(i)| × 2³²  |  SHA-1: sqrt of primes
Source Code
Python — MD5 Implementation 
def r(x, c): return ((x << c) | (x >> (32 - c))) & 0xffffffff

def md5(m):
    m = bytearray(m, 'utf-8'); l = (8 * len(m)) & 0xffffffffffffffff
    m += b'\x80'
    while len(m) % 64 != 56: m += b'\x00'
    m += l.to_bytes(8, 'little')

    A,B,C,D = 0x67452301,0xefcdab89,0x98badcfe,0x10325476
    K = [int(abs(__import__('math').sin(i+1))*2**32)&0xffffffff for i in range(64)]
    s = [7,12,17,22]*4+[5,9,14,20]*4+[4,11,16,23]*4+[6,10,15,21]*4

    for i in range(0, len(m), 64):
        X = [int.from_bytes(m[i+j:i+j+4], 'little') for j in range(0, 64, 4)]
        a, b, c, d = A, B, C, D
        for j in range(64):
            if j<16:   f=(b&c)|(~b&d);     g=j
            elif j<32: f=(d&b)|(~d&c);     g=(5*j+1)%16
            elif j<48: f=b^c^d;            g=(3*j+5)%16
            else:       f=c^(b|~d);         g=(7*j)%16
            a,b,c,d = d,(r((a+f+K[j]+X[g])&0xffffffff,s[j])+b)&0xffffffff,b,c
        A=(A+a)&0xffffffff; B=(B+b)&0xffffffff
        C=(C+c)&0xffffffff; D=(D+d)&0xffffffff
    return ''.join(x.to_bytes(4, 'little').hex() for x in [A, B, C, D])

t = input("Text: ")
print("MD5:", md5(t))

Conclusion
MD5 is still widely used for non-security purposes like file checksums and database deduplication. However, it is cryptographically broken: collision attacks are computationally feasible (demonstrated in 2004), and rainbow table attacks can reverse common password hashes. For security applications, use SHA-256/SHA-3 or bcrypt/Argon2 for passwords.
Experiment 07 — Key Exchange
Diffie-Hellman Key Exchange
A method for two parties to establish a shared secret over an insecure channel without prior communication. Introduced by Whitfield Diffie and Martin Hellman in 1976 — the first public-key protocol.
Key ExchangeDiscrete LogPublic ChannelNo Pre-shared Secret
Aim
To implement the Diffie-Hellman key exchange protocol and demonstrate how two parties can independently compute an identical shared secret without ever transmitting it.
Protocol Steps
  1. Agree publicly on a large prime p and generator g (primitive root mod p)
  2. Alice selects private key a, computes public value A = g^a mod p
  3. Bob selects private key b, computes public value B = g^b mod p
  4. Both A and B are exchanged over the public channel
  5. Alice computes shared secret: S = B^a mod p
  6. Bob computes shared secret: S = A^b mod p — both arrive at identical S
Source Code
Python 
p = int(input("P: "))
g = int(input("G: "))
a = int(input("Alice key: "))
b = int(input("Bob key: "))

A = pow(g, a, p)   # Alice's public value
B = pow(g, b, p)   # Bob's public value

print("Public:", A, B)

s1 = pow(B, a, p)  # Alice computes shared secret
s2 = pow(A, b, p)  # Bob computes shared secret

print("Secret:", s1, s2)
print("Success!" if s1 == s2 else "Failed")

Conclusion
Diffie-Hellman solves the key distribution problem elegantly: a shared secret can be established over a public channel. Security relies on the Discrete Logarithm Problem — easy to compute g^a mod p, computationally infeasible to recover a from A and p for large primes. DH is vulnerable to man-in-the-middle attacks without authentication; this is addressed in authenticated variants like DHE within TLS.
