Experiment 06 — Cryptographic Hash
MD5 Hash Function
MD5 (Message Digest Algorithm 5) produces a 128-bit hash. Designed by Ronald Rivest in 1991. Widely used for checksums and file integrity verification despite known cryptographic weaknesses.
One-Way128-bit OutputLittle-EndianBroken
Aim
To implement MD5 and compare its structure with SHA-1, understanding the use of sine-derived constants, four auxiliary functions, and the little-endian byte order.
MD5 vs SHA-1 Comparison
Output Size
MD5: 128 bits (32 hex)  |  SHA-1: 160 bits (40 hex)
Rounds
MD5: 64 rounds (4×16)  |  SHA-1: 80 rounds (4×20)
Byte Order
MD5: Little-Endian  |  SHA-1: Big-Endian
Constants
MD5: |sin(i)| × 2³²  |  SHA-1: sqrt of primes

Python — MD5 Implementation 
def r(x, c): return ((x << c) | (x >> (32 - c))) & 0xffffffff

def md5(m):
    m = bytearray(m, 'utf-8'); l = (8 * len(m)) & 0xffffffffffffffff
    m += b'\x80'
    while len(m) % 64 != 56: m += b'\x00'
    m += l.to_bytes(8, 'little')

    A,B,C,D = 0x67452301,0xefcdab89,0x98badcfe,0x10325476
    K = [int(abs(__import__('math').sin(i+1))*2**32)&0xffffffff for i in range(64)]
    s = [7,12,17,22]*4+[5,9,14,20]*4+[4,11,16,23]*4+[6,10,15,21]*4

    for i in range(0, len(m), 64):
        X = [int.from_bytes(m[i+j:i+j+4], 'little') for j in range(0, 64, 4)]
        a, b, c, d = A, B, C, D
        for j in range(64):
            if j<16:   f=(b&c)|(~b&d);     g=j
            elif j<32: f=(d&b)|(~d&c);     g=(5*j+1)%16
            elif j<48: f=b^c^d;            g=(3*j+5)%16
            else:       f=c^(b|~d);         g=(7*j)%16
            a,b,c,d = d,(r((a+f+K[j]+X[g])&0xffffffff,s[j])+b)&0xffffffff,b,c
        A=(A+a)&0xffffffff; B=(B+b)&0xffffffff
        C=(C+c)&0xffffffff; D=(D+d)&0xffffffff
    return ''.join(x.to_bytes(4, 'little').hex() for x in [A, B, C, D])

t = input("Text: ")
print("MD5:", md5(t))

Conclusion
MD5 is still widely used for non-security purposes like file checksums and database deduplication. However, it is cryptographically broken: collision attacks are computationally feasible (demonstrated in 2004), and rainbow table attacks can reverse common password hashes. For security applications, use SHA-256/SHA-3 or bcrypt/Argon2 for passwords.
