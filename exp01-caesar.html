Experiment 01 — Classical Cipher
Caesar Cipher
A substitution cipher in which each letter of the plaintext is replaced by a letter a fixed number of positions down the alphabet. Named after Julius Caesar, who used it in his private correspondence.
Symmetric Substitution Monoalphabetic O(n) Time
Aim
To implement and demonstrate the Caesar Cipher encryption and decryption technique, and to understand the concept of substitution ciphers and their vulnerabilities to frequency analysis.
Objectives
  • Understand the concept of shift-based substitution in classical cryptography
  • Implement both encryption and decryption using modular arithmetic
  • Analyze the cipher's vulnerability to brute-force attacks (only 25 possible keys)
  • Observe how case preservation is handled in implementation
Theory
Encryption Formula
E(x) = (x + n) mod 26
Decryption Formula
D(x) = (x − n) mod 26
Key Space
26 possible shifts (0–25). Shift 0 = identity. Trivially broken by brute force.
Historical Use
Julius Caesar used shift 3. ROT13 (shift 13) is a modern symmetric variant.
Algorithm
  1. Accept plaintext input and integer shift value s from the user
  2. Reduce s modulo 26 to normalize (e.g., shift 27 → shift 1)
  3. For each character: if uppercase, apply (ord(c) − 65 + s) % 26 + 65
  4. If lowercase, apply (ord(c) − 97 + s) % 26 + 97; non-alpha chars pass unchanged
  5. Concatenate transformed characters to produce ciphertext
  6. To decrypt, apply the same function with shift −s (or equivalently 26 − s)
Source Code
Python 
def c(t, s):
    return ''.join(
        chr((ord(x) - 65 + s) % 26 + 65) if x.isupper() else
        chr((ord(x) - 97 + s) % 26 + 97) if x.islower() else x
        for x in t
    )

t = input("Text: ")
s = int(input("Shift: ")) % 26
e = c(t, s)

print("Encoded:", e)
print("Decoded:", c(e, -s))

Conclusion
The Caesar Cipher demonstrates the fundamental concept of substitution encryption. While historically significant, it is trivially broken by brute-force (only 25 non-trivial keys) or frequency analysis (letter frequency in ciphertext matches natural language). It serves as the basis for understanding more complex polyalphabetic and transposition ciphers.
