Experiment 07 — Key Exchange
Diffie-Hellman Key Exchange
A method for two parties to establish a shared secret over an insecure channel without prior communication. Introduced by Whitfield Diffie and Martin Hellman in 1976 — the first public-key protocol.
Key ExchangeDiscrete LogPublic ChannelNo Pre-shared Secret
Aim
To implement the Diffie-Hellman key exchange protocol and demonstrate how two parties can independently compute an identical shared secret without ever transmitting it.
Mathematical Foundation
Public Parameters
Large prime p, generator g
Security Basis
Discrete Logarithm Problem: easy to compute g^a mod p, infeasible to reverse
Alice's Computation
A = g^a mod p  →  S = B^a mod p
Bob's Computation
B = g^b mod p  →  S = A^b mod p
Protocol Steps
  1. Agree publicly on a large prime p and generator g (primitive root mod p)
  2. Alice selects private key a, computes public value A = g^a mod p
  3. Bob selects private key b, computes public value B = g^b mod p
  4. Both A and B are exchanged over the public channel
  5. Alice computes shared secret: S = B^a mod p
  6. Bob computes shared secret: S = A^b mod p — both arrive at identical S
Source Code
Python 
p = int(input("P: "))
g = int(input("G: "))
a = int(input("Alice key: "))
b = int(input("Bob key: "))

A = pow(g, a, p)   # Alice's public value
B = pow(g, b, p)   # Bob's public value

print("Public:", A, B)

s1 = pow(B, a, p)  # Alice computes shared secret
s2 = pow(A, b, p)  # Bob computes shared secret

print("Secret:", s1, s2)
print("Success!" if s1 == s2 else "Failed")

Conclusion
Diffie-Hellman solves the key distribution problem elegantly: a shared secret can be established over a public channel. Security relies on the Discrete Logarithm Problem — easy to compute g^a mod p, computationally infeasible to recover a from A and p for large primes. DH is vulnerable to man-in-the-middle attacks without authentication; this is addressed in authenticated variants like DHE within TLS.
